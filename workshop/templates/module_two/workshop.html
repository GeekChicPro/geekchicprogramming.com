Time to Success: 20-25 minutes

You probably thought you knew your computer pretty well, after all, you open it every day to work and play on it. However, the computer that you probably know, is well, a gussied up fancy Word and Hulu machine. We humans love metaphor, and what is the Desktop on your computer if not a series of symbolism that allows us to view our machines as extensions of our work lives? How about the Browser? It is a window into the Internet. Even the term Windows is used metaphorically in computing environments!

There is, however, a lower level than the iconographic metaphors that we're used to. A lower, much more powerful level that uses textual commands to precisely control and modify your computers behavior. You've probably seen it in movies -- it's the Command Line Interface.

With great power comes great responsibility, however. Keep in mind that when using the Command Line, whatever you ask your computer to do, the computer will promptly and without question (most of the time) perform that task, even if that task is binary seppuku.

It is natural, therefore, that we as aspiring programmers, will find ourselves having to familiarize oursleves with that environment. After all, programming languages can also be understood to be precise textual commands that will give you unlimitted freedom in instructing the computer how to operate. And more practically, you'll have to be at least marginally comfortable typing in commands for the rest of the workshop!

Command Syntax

First things first, we have to talk about how to talk to the computer with text.

The Prompt

When you come to the terminal, you'll be presented with something that looks a little like this:

localhost:~ username$

This is called the prompt, and while the details will vary across systems and Terminals, it is communicating some key elements to you, and also prompting you to type in a command. For example, in this case, the localhost tells you the name of the computer that you are controlling (it is possible to command remote computers through the command line), the ~ indicates the directory you're in, in this case ~ is short for "home directory", and the username is the name of the logged in user.

There are other kinds of prompts. One other one is the prompt for the Python REPL interpreter:

>>>

A prompt of this type signals you that you should type in Python commands instead of the shell commands that we'll be learning in this workshop.

Try it! Type in the following commands:

$ python
>>> print "Hello World!"
>>> exit()
$ exit

However, we'll get to Python later. At the moment, we need to familiarize ourselves with the shell -- the environment used to control the computer.

Anatomy of a Command

Here is an example command (go ahead and try it!)

$ ls -la ~/Desktop

This command has several parts:

  • $ The prompt, an indication that you're in the correct place. When we discuss commands I will include the prompt, but you don't have to type it.
  • ls This is the command, also called a Utility. These are programs that are stored on your computer that when executed will perform various tasks based on the input (flags and arguments) and output the results in text to you.
  • -la Are flags, two flags in particular, the -l flag and the -a flag. Flags are options or preferences to the command, more on this later.
  • ~/Desktop This is an argument to the Utility. Arguments are used to specify ordered information that the command needs to function. You can think of them as input. Arguments are usually placed at the end of the command, and there can be any number of space separated arguments from none to many.

A note on flags: flags come in two flavors: single letter flags as in -l or whole word flags such as --version. In the single letter version you can chain lots of flags together after the dash, however in the whole word flag, you have to space seperate the flags. Note that -la is the same as -l -a generally speaking.

In this particular case, the ls utility lists the contents of a directory and the -l flag asks for more information, while the -a asks to see all files, even hidden ones. the ~/Desktop specifies what directory to show the listing of.

Try to run the ls utility without the -l or -a flags.

Run the command without the ~/Desktop argument.

Some Commands to Memorize

Here are a list of the basic utilities on Linux/Unix Terminals that you will use on a regular basis. They are worth comitting to memory! Anything with all capital letters that starts with a dollar sign, like $ARG is an argument to give to the command. Replace $ARG with the value you'd like to pass it.

man $COMMAND
Manual. Get information on how to use the command! Press up and down to scroll through the manual, press Q to quit and go back to the prompt.
ls $DIR
List the contents of the directory $DIR. If no argument is specified will list the contents of the current working directory.
touch $FILE
Creates a file at the specified $FILE path or modifies an existing file's meta data concerning it's modified and accessed time.
mkdir $DIR
Creates a directory in the current working directory.
rmdir $DIR
Removes an empty directory.
cd $DIR
Change directory. Changes the current working directory to the $DIR directory argument. Basically allows you to move around the computer.
pwd
Print Working Directory. Lets you know where you are on the computer.
less $FILE
Displays the contents of a file. Use the up and down arrows to scroll. Press Q to go back to the prompt.
cp $FILE $DESTINATION
Copy. Copies the file to the destination.
mv $FILE $DESTINATION
Move. Moves the file to the destination.
rm $FILE
Remove. Deletes a file permanently - there is NO way to get it back. Be careful when using this!
sudo $CMD
Super User Do. Append sudo before a command and it will prompt you for your password. When entered, this tells the computer that it should execute the command as though it were the super administrator, and will allow you to do anything. Be careful when using this!

Try it out! Execute the following commands (in order):

$ man ls
$ touch testfile.txt
$ ls -l
$ cp testfile.txt testfile2.txt
$ ls -l
$ mv testfile.txt testfile3.txt
$ ls -l
$ rm testfile3.txt
$ ls -l
$ pwd
$ mkdir testdir
$ cd testdir
$ pwd
$ ls -l
$ cd ..
$ pwd
$rm testfile2.txt

Computer Organization

Computers are organized according to a hierarchical directory file structure. Every computer has a root directory that contains every other folder and directory. In Linux this is simply called /, whereas in Windows it's usally called C:\ but keep in mind, with Cygwin, we'll be only talking about the Unix and Linux paradigm.

Because of this structure, every file and directory has an associated absolute path - that is how to find it from the root directory. As you go down from the root directory, the path gets longer, as you go up towards the root directory, it gets shorter. You should recognize this from using Finder or Explorer to navigate your computer's directory structure.

Additionally there is the notion of the relative path - that is, the path to a file or directory from where you currently are in the file system. Again, this is like double clicking a folder in Finder or Explorer -- you're in a particular directory and you can go up to it's parent folder or down into child folders.

absolute paths always start with / relative paths never start with /

Additionally there are two special symbols: . and .. that represent your current working directory and your parent directory respectively. These are often used in relative paths to more clearly specify direction.

Try it out! Type in the following:

$ cd ~
$ pwd
$ ls $PWD/Documents/

$PWD is the output of the pwd command

$ ls ./Documents/

Navigating the Filesystem

You're already pretty well equipped to navigate the filesystem -- go ahead and take a look around! Use the cd command to change directories, and use the pwd command if you get lost.

Use cd .. to go up a parent directory.

Use cd ~ as a shortcut to get back to your home directory.

Remember to use relative and absolute paths as you work your way around.

Pro Tips

Here are some shortcuts to make your life on the command line a bit easier (less typing!)

Tab Autocomplete: When typing out a path or a command, you can quickly finish the rest by pressing the tab key after you type two or three letters. If more than one possibility exists, nothing will happen, but if you hit tab twice, it will display all the posibilities. This will make you much, much faster!

Previous Commands: Instead of retyping an entire command that you made a type on, hit the up and down arrows to go through your command history!

Final Warnings

Again, when interacting with the computer via the command line, you will have an incredible amount of power to ruin your computer and never get your data back. If you're unsure, don't issue a command!.
