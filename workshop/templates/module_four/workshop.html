Time to Success: 45-60 minutes

In this module we're going to teach you how to write again. The first time you learned how to write, you learned about the structure of documents: an introduction, the body, and then a conclusion. Structure is so important to the understanding of documents that we create specialized headings and sections to better break down a large piece of content for the consumption of our human audience. Generally speaking, the way that we format those typographic elements gives visual cues to our readers about how best to approach these sections, for example a short statement that is big and bold is probably a chapter heading.

Unfortunately, computers do not understand visual formating clues to interpret documents, and it is just as important for computers to understand human documents as it is for humans to do so- so that they can render them in a way we can understand. In order for a computer to understand the structure of a document, we use something called a markup language, and in particular we will be dealing with HTML5 in this workshop.

So today's task will be to get you to seperate structure from format in a document, and along the way, we'll create an HTML profile page for you.

A Basic Web Document

All HTML5 web pages are required to have the following elements:

  • A doctype, for HTML5 this is: <!DOCTYPE html>
  • A single <html> tag -- all other tags will be inside this tag.
  • A single <head> and <body> as the two children of the <html> tag
  • A <title> tag inside of the <head> tag.

Open up your basic text editor and type in the following:


 <!DOCTYPE html>
 <html>
 <head>
    <title>My Awesome Profile</title>
 </head>
 <body>

 </body>
 </html>

Save the document on your desktop as "profile.html" - note that if you have Windows or an editor that defines the file type, set the type as "All Files" or "HTML".

Double click the profile.html file on your desktop. It should automatically open a Browser window... with nothing in it. But hey- look! You're title is at the top of the browser tab! If you had trouble opening the file, please let an instructor know. To fix this you can right-click and select "open with" and select a Browser. If it opens in a text editor, it is probably saved as "profile.html.txt"

Let's fix the emptiness. Inside of the <body> tag, insert the following:


<body>
    <h1>My Awesome Profile Page!</h1>
</body>

Reload the page by pressing CTRL + R or CMD + R or F5 or by simply hitting the refresh button. A big, bold version of "My Awesome Profile Page!" should appear.

A few discussion points

As you can see, the basic structure of any HTML page is a head and a body. The head describes the document, including things like its title, and many other useful things as well. The body contains the content of the document. This is starting to seem pretty familiar!

A few other things you can put in the <head> tag are things called <meta> tags -- these tags describe the document to the computer, including things like encoding, language, authorship, descriptions, and more. These tags are used in particular by search engines to categorize the Internet.

Add the following in the <head> section of your document:

 
 <head>
    <meta charset="utf-8">
    <title>My Awesome Profile Page!</title>
    <meta name="author" content="Me">
    <meta name="keywords" content="profile, me, awesome">
    <meta name="description" content="All about me!">
 </head>

There, now Google can find you! This is so common, that web developers have standardized the template for HTML5 docs, the best one is the HTML5 Boilerplate - but this is a bit advanced, so don't bounce over there quite yet!

Adding Content

Ok, now on to the <body> -- let's get some stuff into our awesome profile!

Note - I'm merely making suggestions here, add as much content as you'd like to, not just what I'm suggesting! See the W3C Tag Reference for a complete tag reference if you need a little inspiration.

First add some sections:


 <h2>About Me</h2>
 <h2>My Favorite Things</h2>
 <h3>Music</h3>
 <h3>Movies</h3>

You'll notice when you refresh the page that your Browser renders these headings like you'd expect. Both you and the computer are able to detect the structure of the document with just a little markup!

Add some <p> tags with a short biography under the About section.

Let's also add some links to our social media:

 <p><strong>Follow me:</strong></p>

 <ul>
    <li><a href="http://twitter.com/username">Twitter</a></li>
    <li><a href="http://facebook.com/username">Facebook</a></li>
    <li><a href="http://linkedin.com/in/username">Linked In</a></li>
 </ul>

List your favorite music and movies in those sections using <ol> tags to create ordered lists.

A bit more discussion

All this markup is read by the Browser and is translated into what you're seeing on the browser page, in fact - different browsers will treat the markup slightly differently. If you have a different Web Browser installed, try opening your document up in that one and seeing the difference! Web designers have to use styles to apply universal formatting to their documents, we'll get into that later.

Also, since every web page is essentially markup, most browsers will allow you to see the raw tags. In Chrome, for instance, right click on the page and click "Inspect Element" this will bring up a tab with the raw HTML that's underneath. You can inspect ANY webpage this way, but beware, it can be a jungle of markup out there!

Ask your instructor how to view the raw HTML in your browser.

A Profile Picture

Images and other resources like stylesheets, scripts, and even sounds and music can be linked to and added as resources in web documents. You can use the file structure to have the browser automatically fetch these images- but we're going to use a resource on the Internet - a Globally Recognized Avatar (Gravatar) that you can manipulate and upload without worrying about server resources.

Go to https://en.gravatar.com/ and sign in or create a new account. Select your gravatar or upload a new photo of yourself. Once you have assigned the Gravar to your email- select the "link" button in the My Emails page and copy the link, which should look something like:

wwww.gravatar.com/avatar/5dad8ad2a5bd5773bc08edeea9b27aa1.png

Add the gravatar to your profile:


 <img src="http://wwww.gravatar.com/avatar/5dad8ad2a5bd5773bc08edeea9b27aa1.png" alt='Profile' title='Me' />

If you're having trouble with Internet connections, simply move an image into the same working directory as the profile.index file. Then simply add the name of the image without any path information to the src attribute.

Styling it Up with CSS (and Bootstrap)

Actual format of web documents is done using CSS (Cascading Style Sheets). Although a full discussion on CSS is beyond the scope of this workshop; here's a bit of a preview.

 <style type="text/css">
     body {
        background-color: white;
     }

     p {
        color: black;
        font-size: 16px;
        font-family: serif;
        line-height: 18px;
     }

     ul.horizontal li {
         display: inline;
     }
 </style>

Essentially what CSS does is allow you to specify exactly how each of our HTML elements should look by laying out formating properties for each of the elements. Additionally, CSS allows you to specify classes (that's the .horizontal after the <ul> tag) to specifically style a set of elements. To add a class to your elements, set it in the class attribute:


 <ul class="horizontal">
    <li>List Item One</li>
    <li>List Item Two</li>
 </ul>

The best place to put <style> tags is in the head of your html document. You can also link to them as external documents, or add styles directly to tags with the style attribute. In fact, this is where the term Cascading comes from. Styles are applied in this precedence: styles on the tag, styles in the document, external style documents.

Bootstrap

Rather than get you to write CSS, I highly recommend using the Bootstrap framework to quickly add an elegant, and universal style to your web documents. You may notice that these pages are also in Bootstrap!

Add the following to your <head> block, after all the metas:


 <link rel="stylesheet" type="txt/css" 
    href="//netdna.bootstrapcdn.com/twitter-bootstrap/2.3.1/css/bootstrap-combined.min.css" />

When you refresh your page, everything should look different! You can now add Bootstrap markup to your document to more specifically control what is happening. Let's add the following markup to center your document on the page and clean things up a bit:


 <body>
    
    <div class="container">
        <div class="row">
            <div class="span12">

               [Your content here]

            </div>
        </div>
    </div>

 </body>

Feel free to use the excellent documents from Bootstrap to continue to add markup to your page that will format it.

A note on Interactivity

The final piece that is is missing with front end development is some sort of interactivity. Web documents have moved beyond simple information giving things to full blown applications. Because of this some user interaction with the document is required. To accomplish this, we use JavaScript.

Again, JavaScript is beyond the scope of this workshop, but if you do intend to become a front end web developer, then that trinity - HTML5, CSS, and JavaScript will be your bread and butter.

To very quickly and easily develop JavaScript on all Browsers, we use a library called jQuery - a selector engine for your document. To try it out, add the following to your document, in the <head> block after the style sheet link.


 <head>
    
    ...

    <script type="text/javascript" src="http://code.jquery.com/jquery-1.9.1.min.js"></script>
    <script type="text/javascript">
        
        $(document).ready(function() {
            $("p").click(function() {
                $(this).toggleClass('text-success');
            });
        });

    </script>

 </head>

Now go click on some of your paragraphs -- they should turn green when you click them, and then back to black when you click on them again! (This step also assumes you have the Bootstrap stylesheet added to your document).
